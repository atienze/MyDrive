Edit `refreshData()` and `loadFilesViewData()` in dashboard.html to decouple server file loading
from the global refresh cycle.

1. **Rewrite `refreshData()`** — fetch only two endpoints (drop `/api/files/server`):
   ```javascript
   async function refreshData() {
     try {
       const [statusRes, clientRes] = await Promise.all([
         fetch('/api/status'),
         fetch('/api/files/client'),
       ]);
       if (!statusRes.ok || !clientRes.ok) throw new Error('non-2xx');
       const [snapshot, clientData] = await Promise.all([
         statusRes.json(),
         clientRes.json(),
       ]);
       const clientFiles = clientData.files || [];
       // usedBytes: use cached server map if available, else 0
       const usedBytes = Object.values(App.cachedServerMap).reduce((acc, f) => acc + (f.size || 0), 0);
       hideErrorBanner();
       renderOverview(snapshot, usedBytes);
       loadFilesViewData(clientFiles);
     } catch (_) {
       showErrorBanner('Server unreachable \u2014 click Refresh to retry');
     }
   }
   ```
   Note: `usedBytes` for the donut chart now comes from `App.cachedServerMap` (populated lazily
   by the Server tab Refresh) — this means the donut shows 0 until the user visits the Server tab
   at least once. This is acceptable behavior per SERV-02 (no auto server fetch).

2. **Update `loadFilesViewData()`** — remove the `serverFiles` parameter; no longer builds
   `cachedServerMap` here (that's now done by `loadServerViewData()` in plan 02):
   ```javascript
   function loadFilesViewData(clientFiles) {
     App.cachedFilesViewData = clientFiles || [];
     App.cachedClientMap = Object.create(null);
     for (const f of App.cachedFilesViewData) App.cachedClientMap[f.rel_path] = f;
     renderBreadcrumb();
     renderFilesTable();
   }
   ```
   Important: `App.cachedServerMap` is no longer updated here — it keeps whatever value it had
   (empty on first load, populated after user visits Server tab). This means local view sync dots
   start as amber and update to synced after the user loads the Server view at least once.
   This is correct behavior — the local view sync dot reflects "is this file also on the server?"
   which requires the server list to have been loaded at least once.

3. **Verify no other call sites pass two args to `loadFilesViewData`** — check the file for any
