  Three changes to dashboard.html:

  **1. Add search input HTML in the header** — Between the left `<div>` (containing menu-btn + header-title) and the right `<div>` (containing stats/status/sync-btn), insert:

  ```html
  <div class="flex-1 max-w-xs mx-4 hidden sm:block">
      <input type="search" id="search-input" placeholder="Filter files..."
             oninput="updateSearch(this.value)"
             class="w-full bg-[#0f1a2e] border border-[#2a2a4a] rounded-md px-3 py-1.5 text-sm text-[#d0d8f0] placeholder-[#556] focus:outline-none focus:border-[#4d9eff44]">
  </div>
  ```

  The input is `hidden sm:block` — hidden on very small screens (where header space is tight), visible from `sm` breakpoint up. On mobile, the bottom nav provides navigation and the file list is short enough to scroll. If testing shows there is room on mobile, change to `block` (this is a Claude's discretion item from the research).

  **2. Add `searchQuery` variable and `updateSearch()` function** — In the `<script>` block, near the other global state variables (around line 158), add:

  ```javascript
  var searchQuery = '';

  function updateSearch(val) {
      searchQuery = val.trim().toLowerCase();
      rerenderLocal();
      rerenderServer();
  }
  ```

  **3. Integrate search filter into rerender functions** —

  In `rerenderLocal()`, immediately after `var entries = getEntriesAtPath(cachedClientFiles, localCurrentPath);`, add:
  ```javascript
  if (searchQuery) {
      entries = { folders: [], files: entries.files.filter(function(f) {
          return f.rel_path.toLowerCase().indexOf(searchQuery) !== -1;
      }) };
  }
  ```

  In `renderServerGroups()`, inside the `deviceNames.forEach` loop, immediately after `var entries = getEntriesAtPath(groups[device], devicePath);`, add the same filter:
  ```javascript
  if (searchQuery) {
      entries = { folders: [], files: entries.files.filter(function(f) {
          return f.rel_path.toLowerCase().indexOf(searchQuery) !== -1;
      }) };
  }
  ```

  Important: Do NOT mutate the original `entries` object from `getEntriesAtPath` — create a new object with filtered results. This ensures clearing search restores all files.

  **4. Update empty state during search** — In `rerenderLocal()`, when `!html` and search is active, show a "No matching files" message instead of the standard empty state:
  ```javascript
  if (!html) {
      if (searchQuery) {
          localList.innerHTML = '<div class="col-span-full py-6 text-center text-[#556] text-[13px]">No files matching "' + escapeHtml(searchQuery) + '"</div>';
      } else if (cachedClientFiles.length === 0) {
          localList.innerHTML = emptyStateHtml('local');
      } else {
          localList.innerHTML = '<div class="col-span-full py-6 text-center text-[#556] text-[13px]">This folder is empty</div>';
      }
  }
  ```

  Similarly in `renderServerGroups()`, after rendering all device groups, if the total rendered card count is 0 and searchQuery is active, show a search-empty message.
